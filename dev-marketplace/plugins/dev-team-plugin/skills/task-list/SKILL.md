# 작업 목록 생성 스킬

## 출력 템플릿

```markdown
# [기능명] 작업 목록

## 에픽(Epic): [대분류 기능]

### 스프린트 1
- [ ] [BE] 작업명: 상세 설명 (예상: Xh)
- [ ] [FE] 작업명: 상세 설명 (예상: Xh)
- [ ] [DB] 작업명: 상세 설명 (예상: Xh)

### 스프린트 2
- [ ] [BE] 작업명: 상세 설명 (예상: Xh)
- [ ] [FE] 작업명: 상세 설명 (예상: Xh)

## 에픽(Epic): [다음 대분류]

### 스프린트 3
- [ ] [BE] 작업명: 상세 설명 (예상: Xh)
- [ ] [FE] 작업명: 상세 설명 (예상: Xh)

---

## 우선순위 분류

### P0 (즉시 착수)
- [핵심 블로커 작업]

### P1 (이번 스프린트)
- [주요 기능 구현 작업]

### P2 (다음 스프린트)
- [개선 및 부가 기능]
```

## 태그 규칙

| 태그 | 대상 |
|------|------|
| `[BE]` | NestJS 서버, API 엔드포인트, 비즈니스 로직 |
| `[FE]` | React/Next.js/RN 컴포넌트, 화면, 상태 관리 |
| `[DB]` | Prisma 스키마, 마이그레이션, 시드 데이터 |
| `[INFRA]` | Docker, CI/CD, 환경 설정 |

## 작성 원칙

- 각 작업은 독립적으로 구현/완료 가능한 단위로 분해
- 예상 시간: 2h 이하 권장, 8h 초과 시 더 분해
- 작업 간 의존성 있으면 스프린트 순서로 표현
- 모든 출력은 한국어로 작성
