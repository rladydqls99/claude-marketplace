# 비판적 코드 리뷰 스킬

## 리뷰 체크리스트

| 영역 | 주요 검토 항목 |
|------|--------------|
| 코드 품질 | 가독성, 중복(DRY), 네이밍, 에러 처리, 매직 넘버 |
| 아키텍처 | SRP 위반, 강결합, 순환 의존성, 계층 위반 |
| 성능 | O(n²) 알고리즘, N+1 쿼리, 메모리 누수, 동기 블로킹 |
| 보안 | SQL Injection/XSS, 인증/인가 누락, 하드코딩 시크릿 |
| 테스트 | 핵심 로직 누락, 엣지 케이스, 부정 케이스 |

## 리스크 수준

| 수준 | 기호 | 기준 |
|------|------|------|
| 치명적 | 🔴 | 보안 취약점, 데이터 손실, 서비스 중단 |
| 높음 | 🟠 | 심각한 성능 저하, 주요 기능 오작동 |
| 중간 | 🟡 | 유지보수성 저하, 잠재적 버그 |
| 낮음 | 🔵 | 코드 스타일, 선택적 개선 |

## 출력 템플릿

```markdown
## 부정적 코드 리뷰 결과

### 리뷰 요약
- **검토 대상**: [파일명/모듈명]
- **발견된 문제 수**: 총 N개 (🔴 치명적: X | 🟠 높음: X | 🟡 중간: X | 🔵 낮음: X)
- **전반적 리스크**: [치명적/높음/중간/낮음]

---

### 🔴 치명적 이슈

#### 1. [이슈 제목]
- **위치**: `파일명.ext` L[라인번호] / `함수명()`
- **문제**: [구체적 문제 설명]
- **리스크**: [야기할 수 있는 결과]
- **개선 방향**:
  ```언어
  // 개선된 코드
  ```

### 🟠 높은 우선순위 이슈
[동일 형식]

### 🟡 중간 우선순위 이슈
[동일 형식]

### 🔵 낮은 우선순위 이슈
[동일 형식]

---

### 액션 아이템
1. [ ] [즉시] [가장 중요한 수정사항]
2. [ ] [단기] [다음으로 중요한 수정사항]
3. [ ] [중기] [개선사항]

### 종합 리스크 평가
[현재 코드를 프로덕션 배포 시 전반적 리스크를 2-3문장으로 요약]
```

## 작성 원칙

- 코드 인용 시 반드시 파일 경로 + 라인 번호 표시
- 모든 지적에 근거(왜 문제인가) + 개선 방향 필수
- 최근 변경된 코드 중심 리뷰 (전체 코드베이스 아님)
- 긍정적 언급은 최소화, 문제 발견에 집중
- 모든 출력은 한국어로 작성
