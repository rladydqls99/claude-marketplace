# PRD 생성 스킬

## 출력 템플릿

```markdown
# [프로젝트/기능명] PRD

## 1. 개요
- **배경 및 목적**: [왜 필요한가]
- **비즈니스 목표**: [달성하려는 목표]
- **성공 지표(KPI)**: [측정 가능한 지표]

## 2. 사용자 정의
- **타겟 페르소나**: [사용자 유형, 특성, 니즈]
- **Pain Point**: [현재 겪는 문제]

## 3. 범위
### In Scope
- [포함 기능 1]
- [포함 기능 2]

### Out of Scope
- [제외 기능 1]

## 4. 기능 명세

### 4.1 [기능명] `[Must Have / Should Have / Nice to Have]`
- **설명**: [기능 설명]
- **사용자 스토리**: "[사용자]로서, [목표]를 위해 [기능]을 원합니다."
- **수락 기준(Acceptance Criteria)**:
  - [ ] [검증 가능한 조건 1]
  - [ ] [검증 가능한 조건 2]
- **엣지 케이스**: [예외 상황]

## 5. 비기능 요구사항
- **성능**: [응답 시간, 처리량]
- **보안**: [인증, 권한, 데이터 보호]
- **확장성**: [확장 고려사항]

## 6. 의존성 및 제약사항
- [외부 시스템, API, 팀 간 의존성]
```

## 우선순위 기준

| 등급 | 기준 |
|------|------|
| Must Have | 없으면 제품 출시 불가 |
| Should Have | 중요하나 초기 버전 없어도 가능 |
| Nice to Have | 있으면 좋지만 나중에 추가 가능 |

## 작성 원칙

- 수락 기준은 반드시 검증 가능한 형태로 작성
- 기술 용어는 한국어 + 영문 병기: 인증(Authentication)
- 불명확한 요구사항은 질문(한 번에 3개 이하)
- 모든 출력은 한국어로 작성
