# 긍정적 코드 리뷰 스킬 (Positive Review)

코드에서 잘 된 점, 모범 사례, 칭찬할 만한 패턴을 발굴한다.
단순한 칭찬이 아니라 **왜 좋은지** 근거와 함께 제시한다.

## 리뷰 관점

### 1. 아키텍처 & 구조
- FSD 레이어 분리가 올바른지 (단방향 의존성 준수)
- 단일 책임 원칙(SRP) 준수 여부
- 재사용 가능한 추상화 수준이 적절한지

### 2. 성능 최적화
- 불필요한 리렌더링 방지 (memo, useMemo, useCallback 적절한 사용)
- 비동기 처리 효율성 (병렬 처리, 중복 요청 제거)
- 번들 최적화 (동적 import, 코드 스플리팅)

### 3. 코드 품질
- 타입 안전성 (TypeScript 활용도)
- 에러 핸들링 및 엣지 케이스 처리
- 가독성과 명확한 네이밍

### 4. 사용자 경험
- 로딩 상태, 빈 상태, 에러 상태 처리
- 접근성(aria, 키보드 내비게이션)
- 반응형 처리

### 5. 유지보수성
- 테스트 가능한 구조
- 주석 및 문서화 품질
- 일관된 코딩 컨벤션

## 출력 템플릿

```markdown
# ✅ 긍정적 코드 리뷰

## 총평
[코드 전체에 대한 긍정적 총평 2-3문장]

---

## 🌟 탁월한 부분

### [항목명]
**위치**: `파일명:라인번호` 또는 컴포넌트명
**내용**:
\`\`\`tsx
// 해당 코드 발췌
\`\`\`
**왜 좋은가**: [구체적인 이유. 어떤 문제를 해결하는지, 왜 이 패턴이 적절한지]

---

## 👍 잘 된 부분

### [항목명]
- **위치**: `파일명`
- **내용**: [설명]
- **근거**: [왜 좋은지]
```

## 작성 원칙

- 막연한 칭찬("코드가 깔끔하네요") 금지 — 반드시 구체적인 근거 제시
- 코드 발췌 + 이유 세트로 작성한다
- 긍정적 리뷰지만 객관적이어야 한다 — 과장 금지
- 재사용 가능한 패턴은 따로 추출해 팀 전체가 참고할 수 있게 한다
- 모든 출력은 한국어로 작성한다
