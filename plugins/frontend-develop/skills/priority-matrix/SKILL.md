# 우선순위 분류 스킬 (Priority Matrix)

기능 목록 또는 태스크 목록을 사용자 가치 × 구현 복잡도 기준으로 분류한다.
모든 요구사항을 P0~P2로 구분하고, 개발 순서와 MVP 범위를 명확히 제시한다.

## 우선순위 기준

### P0 — Must Have (즉시 착수, MVP 필수)
- 서비스가 동작하기 위해 반드시 필요한 기능
- 없으면 사용자가 핵심 가치를 얻을 수 없는 기능
- 비즈니스 요구사항 또는 법적 의무사항

### P1 — Should Have (이번 릴리즈, 중요)
- 있으면 사용자 경험이 크게 향상되는 기능
- 경쟁 서비스 대비 필수적인 기능
- 사용자 불만이 발생할 수 있는 결핍 요소

### P2 — Nice to Have (다음 스프린트, 개선)
- 있으면 좋지만 없어도 서비스 운영 가능
- 고급 사용자를 위한 기능
- 성능/UI 개선, 분석 도구 연동 등

## 분류 방법

각 기능/태스크에 대해 아래 두 축으로 판단한다:

```
사용자 가치 (높음↑)
    |  [P0]         [P1]
    |  Must Have    Should Have
    |
    |  [P0]         [P2]
    |  블로커       Nice to Have
    +------------------------→
       구현 복잡도 (낮음←→높음)
```

## 작성 원칙

- 우선순위 결정 이유를 반드시 명시한다 (이유 없는 분류 금지)
- 기획자의 요구를 무조건 P0으로 받지 않고, 사용자 가치와 복잡도로 판단한다
- 리스크(기술적 부채, 의존성, 외부 API 등)를 별도 항목으로 표시한다
- 총 예상 공수를 계산해 현실적인 일정인지 검토한다
- 모든 출력은 한국어로 작성한다
