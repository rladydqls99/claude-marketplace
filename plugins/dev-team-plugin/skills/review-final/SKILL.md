# 최종 리뷰 통합 스킬

## 배포 판정 기준

| 판정 | 조건 |
|------|------|
| APPROVED | Critical 이슈 없음, High 이슈 없거나 허용 수준, 필수 체크리스트 전항목 통과 |
| CONDITIONAL | Critical 없음, High 일부 존재하나 수정 후 배포 가능 |
| REJECTED | Critical 이슈 1개 이상, 필수 요구사항 미충족, 보안/데이터 위험 존재 |

## 우선순위 분류

| 등급 | 정의 | 배포 영향 |
|------|------|----------|
| 🔴 Critical | 기능 불가, 보안 취약점, 데이터 손실, 테스트 실패 | 배포 차단 |
| 🟠 High | 성능 저하, 중요 기능 결함 | 조건부 배포 |
| 🟡 Medium | 코드 품질, 가독성 개선 | 배포 후 추적 |
| 🟢 Low | 향후 개선 제안, 스타일 | 백로그 |

## 출력 템플릿

```markdown
## 최종 리뷰 리포트

### 최종 판정
**[APPROVED / CONDITIONAL / REJECTED]** — [한 줄 요약 근거]

---

### 리뷰 통합 요약

#### 긍정적 리뷰 핵심
- [강점 1]
- [강점 2]

#### 부정적 리뷰 핵심
- [이슈 1]
- [이슈 2]

#### 상충 의견 조율
- [상충 사항]: [조율 결과 및 근거]

---

### 우선순위별 액션 아이템

#### 🔴 Critical (즉시 수정 필수)
- [ ] [액션]: [구체적 조치 방법]

#### 🟠 High (배포 전 수정 권장)
- [ ] [액션]: [구체적 조치 방법]

#### 🟡 Medium (배포 후 개선)
- [ ] [액션]: [목표 시기]

#### 🟢 Low (향후 고려)
- [ ] [액션]: [백로그 추가]

---

### 필수 체크리스트
- [ ] 기능 정상 동작
- [ ] 보안 이슈 없음
- [ ] 성능 기준 충족
- [ ] 모든 테스트 통과

---

### 개선 로드맵
- **단기 (즉시~1주)**: [Critical/High 수정]
- **중기 (1주~1개월)**: [Medium 개선]
- **장기 (1개월~)**: [Low, 아키텍처 개선]

### 최종 의견
[종합적인 판단 근거와 권고사항을 3-5문장으로 서술]
```

## 작성 원칙

- 어느 리뷰어 편도 들지 않고 근거 기반으로 판단
- 리뷰 하나만 제공된 경우: 누락 리뷰 명시 후 가능 범위에서 판정
- 판단 정보 부족 시 가정 명시 또는 추가 정보 요청
- 모든 출력은 한국어로 작성
