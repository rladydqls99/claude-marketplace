# 긍정적 코드 리뷰 스킬

## 리뷰 프로세스

1. `git diff` 또는 PR 내용으로 최근 변경사항 파악
2. 4가지 관점(코드 품질 / 아키텍처 / 성능 / 테스트)으로 강점 발굴
3. 아래 템플릿으로 리뷰 작성

## 출력 템플릿

```markdown
## 긍정적 코드 리뷰 리포트

### 리뷰 요약
[코드의 전반적인 인상과 핵심 강점을 2-3문장으로 요약]

---

### 하이라이트 (Top 3-5)
1. **[강점 제목]** — `파일경로` L[라인번호]
   ```언어
   // 해당 코드 스니펫
   ```
   [왜 훌륭한지 이유]

2. **[강점 제목]** — `파일경로` L[라인번호]
   [설명]

---

### 코드 품질 강점
- **가독성**: [구체적 예시 + 파일:라인]
- **네이밍**: [명확한 변수/함수명 예시]
- **재사용성**: [추상화/컴포넌트화 우수 사례]

### 아키텍처 강점
- **설계 결정**: [관심사 분리, 패턴 적용 등]
- **의존성 관리**: [결합도 최소화 사례]

### 성능 강점 _(해당 시)_
- [효율적인 알고리즘, 캐싱, 비동기 처리 등]

### 테스트 강점 _(해당 시)_
- [커버리지, 엣지 케이스, 테스트 품질]

---

### 이 코드에서 배울 점
- [팀 전체가 참고할 모범 사례 정리]

### 개발자에게
[진심 어린 격려 메시지]
```

## 작성 원칙

- 코드 인용 시 반드시 파일 경로 + 라인 번호 표시
- 막연한 칭찬 금지 — "왜 좋은가" 근거 필수
- 개선 사항은 이 리뷰에서 언급하지 않음 (review-bad 역할)
- 실제 존재하는 강점만 칭찬, 과장 금지
- 모든 출력은 한국어로 작성
