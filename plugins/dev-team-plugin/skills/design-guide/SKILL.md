# 디자인 가이드 생성 스킬

독창적이고 프로덕션급 퀄리티의 디자인 방향을 제시하는 가이드를 생성한다.
뻔한 템플릿 나열이 아니라, 프로젝트의 맥락에 맞는 **구체적인 디자인 결정**을 담는다.

## 핵심 원칙

1. **구체적 결정 > 빈칸 템플릿**: "[색상명]", "[예: ...]" 같은 플레이스홀더 금지. 실제 값을 결정해서 제시한다.
2. **맥락 우선**: 프로젝트 성격, 타겟 사용자, 기존 코드베이스를 파악한 뒤 그에 맞는 디자인을 제안한다.
3. **개성 있는 디자인**: 기본 Bootstrap/Material 느낌을 탈피하고, 프로젝트만의 시각적 아이덴티티를 만든다.
4. **구현 가능성**: Tailwind CSS 클래스, CSS 변수 등 개발자가 바로 적용할 수 있는 형태로 제공한다.

## 작성 프로세스

### 1단계: 맥락 파악
- 프로젝트의 기존 스타일 파일(tailwind.config, globals.css 등)을 확인한다
- 사용자의 요청에서 프로젝트 성격과 분위기를 파악한다
- 타겟 사용자와 사용 환경을 고려한다

### 2단계: 디자인 컨셉 수립
- **무드**: 프로젝트에 어울리는 분위기를 한 문장으로 정의 (예: "따뜻하고 신뢰감 있는 금융 서비스")
- **핵심 키워드 3개**: 디자인 전반을 관통하는 키워드 선정
- **차별점**: 유사 서비스와 구별되는 시각적 특징 1-2가지

### 3단계: 디자인 시스템 정의

아래 항목 중 요청에 필요한 것만 선별해서 제공한다. 모든 항목을 무조건 나열하지 않는다.

**색상**: Primary, Secondary, Accent + 의미색(Success/Error/Warning) + 배경/텍스트. Tailwind 커스텀 색상 설정 코드로 제공.

**타이포그래피**: 폰트 패밀리 추천(웹폰트 기준), 핵심 스케일만 간결하게.

**공간/레이아웃**: 간격 체계, 컨테이너 폭, 그리드 방식을 Tailwind 관점에서 설명.

**컴포넌트 스타일**: 해당 기능에 필요한 컴포넌트만 다룬다. Tailwind 클래스 조합 예시로 제공.

**인터랙션**: 전환/애니메이션은 핵심적인 것만 CSS 코드로 제공.

### 4단계: 적용 예시
- 핵심 화면 1-2개에 대해 실제 Tailwind 클래스를 활용한 JSX 스니펫 제공
- 스니펫은 전체 페이지가 아닌, 디자인 의도를 보여주는 핵심 부분만

## 출력 형식

```markdown
# [기능명] 디자인 가이드

## 컨셉
- **무드**: (한 문장)
- **키워드**: (3개)
- **차별점**: (1-2가지)

## 디자인 시스템
(필요한 항목만 선별 제공 — 색상, 타이포, 레이아웃, 컴포넌트, 인터랙션)

## 적용 예시
(핵심 화면의 JSX + Tailwind 스니펫)
```

## 금지 사항

- HEX 코드만 수십 줄 나열하는 색상표
- 모든 브레이크포인트/접근성/애니메이션을 무조건 포함
- 프로젝트 맥락 없이 범용 디자인 시스템 복붙
- "[예: ...]", "[색상명]" 같은 빈칸 템플릿

## 작성 원칙

- 모든 출력은 한국어로 작성
- 짧고 결정적인 문장 사용 (설명이 아니라 결정을 내린다)
- 코드 예시는 Tailwind CSS 기준으로 제공
- 프로젝트에 이미 있는 설정과 충돌하지 않도록 기존 코드를 먼저 확인
